# 📋 智能分组与专家匹配服务概述

## 服务定位

智能分组与专家匹配服务是科技管理系统的核心服务之一，负责对申报项目进行智能分组，并为每个分组匹配最合适的评审专家。

### 核心能力

1. **智能学科分组**：按学科分类，质量均衡分配
2. **全局最优专家匹配**：以评审组为单元，确保每组有足够的懂行专家
3. **智能关系回避**：自动检测并回避师生、历史合作等关系

### 解决的核心问题

| 现状痛点 | 解决方案 |
|----------|----------|
| 按三级学科分组，同组>30项时无法处理 | 按学科分组，超出则质量评估+均衡分配 |
| 各组质量参差不齐 | LLM评估质量，贪心算法保证均衡 |
| "神仙打架"或专家不懂行 | 全局最优专家匹配 |
| 师生关系、利益冲突 | 智能回避关系检测 |

---

## 功能特性

### 1. 智能分组 (Grouping)

- **学科初步分组**：按项目学科代码(ssxk1)进行初步分组
- **质量评估**：使用 LLM 评估每个项目的创新性、技术难度、应用价值
- **均衡分配**：对超出30项的学科，进行质量均衡分配
- **统计输出**：每组数量、质量分布、组间差异

### 2. 专家匹配 (Matching)

- **专家画像构建**：从研究领域、论文、擅长专业构建专家向量
- **全局最优匹配**：以组为单元的整体优化匹配
- **关系回避**：支持师生、历史合作、单位回避
- **可解释性**：匹配结果附带原因说明

```

│                     智能分组与匹配流程                              │

     用户请求 (year, limit, target_groups=15)

         │
         ▼

├─────────────────────────────────────────────────────────────────┤

         │

                           │ 2. 向量相似度计算   │
                           │ 3. 全局最优匹配    │
                           │ 4. 关系回避检测    │
                           └──────────┬──────────┘
                                      │
                            ┌─────────┴─────────┐
                            │    结果聚合       │

                            │ - 匹配结果       │
                            │ - 可解释报告     │
                            └────────┬─────────┘
                                      │
                                      ▼
                              审查结果输出
```

## 数据来源

### 项目侧数据

| 数据库表 | 字段 | 记录数 | 用途 |
|----------|------|--------|------|
| `Sb_Jbxx` | `xmmc` (项目名称) | ~147,464 | 项目标识 |
| `Sb_Jbxx` | `gjc` (关键词) | - | 主题提取 |
| `Sb_Jbxx` | `ssxk1/2` (学科代码) | - | **学科分组依据** |
| **`Sb_Jj`** | **`xmjj`** (项目简介) | **117,478** | **质量评估内容** |
| `Sb_Jj` | `lxbj` | 103,202 | 补充内容 |

### 学科代码表

| 数据库表 | 字段 | 用途 |
|----------|------|------|
| `hbstanew.sys_subject` | `code` | 学科代码 (如 01, 0101, 010101) |
| `hbstanew.sys_subject` | `name` | 学科名称 |
| `hbstanew.sys_subject` | `parent_id` | 父级ID (用于构建层级) |

**学科层级判断**：
- code 长度=2 → 一级学科 (如 01, 02, 03)
- code 长度=3 → 二级学科 (如 010, 011)
- code 长度≥4 → 三级学科 (如 0101, 0102)

### 专家侧数据

| 数据库表 | 字段 | 用途 |
|----------|------|------|
| `ZJK_ZJXX` | `SXXK1-5` (熟悉学科) | 学科匹配 |
| `ZJK_ZJXX` | `SXZY` (擅长专业) | 领域匹配 |
| `ZJK_ZJXX` | `YJLY` (研究领域) | **核心分析内容** |
| `ZJK_ZJXX` | `LWLZ` (论文论著) | 研究方向推断 |

## 技术选型

| 技术 | 用途 |
|------|------|
| **LangChain LCEL** | Agent 编排、链式调用 |
| **多模态 LLM** | 项目质量评估、专家画像构建 |
| **Qwen text-embedding-v3** | 文本向量化 (专家匹配用) |
| **贪心算法** | 质量均衡分配 |
| **scipy** | 约束优化、线性规划 (专家匹配) |

---

## 分组参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_per_group` | 15 | 每组目标项目数 |
| `split_threshold` | 30 | 超过此数量则拆分该学科 |
| `quality_weights` | 1:1:1 | 创新性:技术难度:应用价值 |
| `quantity_tolerance` | ±3 | 每组数量允许差异 |
| `quality_tolerance` | 10% | 每组质量总分允许差异 |
