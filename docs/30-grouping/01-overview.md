# 📋 智能分组与专家匹配服务概述

## 服务定位

智能分组与专家匹配服务是科技管理系统的核心组件之一。当前重点是：对申报项目做语义分组，并为后续专家匹配提供稳定、可解释的分组结果。

### 核心能力

1. **智能语义分组**：按项目在做什么分组，允许同代码拆分、异代码合并
2. **全局最优专家匹配**：以评审组为单元，确保每组有足够的懂行专家
3. **智能关系回避**：自动检测并回避师生、历史合作等关系

### 解决的核心问题

| 现状痛点 | 解决方案 |
|----------|----------|
| 旧方案把学科代码当硬约束 | 以项目语义为主，LLM主导分组与拆并 |
| 同代码项目主题可能不同 | 允许同代码拆分，不同代码合并 |
| 分组结果难以解释 | 每个组和每个项目都输出理由 |
| 师生关系、利益冲突 | 智能回避关系检测 |

---

## 功能特性

### 1. 智能分组 (Grouping)

- **语义主导分组**：以 `xmmc` 为主，`xmjj` 为辅，判断项目主题
- **LLM 语义判断**：按研究对象、方法、应用场景、目标进行分组
- **容量控制**：对过大组拆分，对过小组逐项目分散归并
- **可解释输出**：记录每个组和每个项目的归组理由与置信度

### 2. 专家匹配 (Matching)

- **专家画像构建**：从研究领域、论文、擅长专业构建专家画像
- **全局最优匹配**：以组为单元的整体优化匹配
- **关系回避**：支持师生、历史合作、单位回避
- **可解释性**：匹配结果附带原因说明

```
┌─────────────────────────────────────────────────────────────────┐
│                     智能分组与匹配流程                          │
└─────────────────────────────────────────────────────────────────┘

     用户请求 (year, limit)
         │
         ▼
┌─────────────────────┐
│   参数校验与预处理   │
└──────┬──────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│                     分组子服务 (Grouping)                       │
├─────────────────────────────────────────────────────────────────┤
│  1. 获取项目名称 + 可选简介                                     │
│  2. 提炼项目主题语义                                            │
│  3. embedding 召回候选组（可选）                                │
│  4. LLM 做主题归属判断                                          │
│  5. 对过小组逐项目重分配，对过大组拆分                           │
│  6. 做代码体系隔离检查                                          │
└────────┬────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────┐     ┌─────────────────────┐
│    分组结果         │     │    专家匹配子服务    │
│ - 主题组             │     │    (Matching)       │
│ - 每组理由           │     │                     │
│ - 复核标记           │     │ 1. 专家画像构建    │
└─────────────────────┘     │ 2. 向量相似度计算   │
                            │ 3. 全局最优匹配     │
                            │ 4. 关系回避检测     │
                            └──────────┬──────────┘
                                      │
                            ┌─────────┴─────────┐
                            │    结果聚合       │
                            │ - 分组结果        │
                            │ - 匹配结果        │
                            │ - 可解释报告      │
                            └────────┬─────────┘
                                      │
                                      ▼
                              审查结果输出
```

## 技术选型

| 技术 | 用途 |
|------|------|
| **LLM** | 项目语义理解、分组决策、结果解释 |
| **Embedding** | 候选组召回、语义相似度筛选 |
| **聚类/优化算法** | 初分组、容量控制、重排 |
| **LangChain LCEL** | Agent 编排、链式调用 |
| **scipy** | 约束优化、线性规划 (专家匹配) |

## 分组参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_per_group` | 5 | 小组下限，低于则继续归并 |
| `max_per_group` | 15 | 每组上限，超过则拆分 |
| `top_k_candidates` | 3 | 每个项目保留的候选组数 |
| `llm_batch_size` | 20 | 单次 LLM 处理项目数 |
| `confidence_threshold` | 0.65 | 低于此值的项目标记复核 |
| `enable_embedding` | true | 是否启用 embedding 召回 |
| `enable_llm` | true | 是否启用 LLM 复核 |
