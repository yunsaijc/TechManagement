# 📋 分组子服务设计

## 概述

分组子服务负责将申报项目按"项目在做什么"进行分组。当前实现采用**学科代码 + 关键词联合分组**的策略：以学科代码形成粗分桶，以项目关键词（`gjc` 字段）计算相似度进行细粒度聚类。

分组的基本原则是：
- 学科代码是分组的第一维度，决定大领域边界
- 关键词是第二维度，决定组内主题一致性
- 同代码但关键词差异大的项目应分到不同组
- 组内主题必须能被人类评审解释
- 小于 `min_per_group` 的组不能原样保留，必须继续归并或拆散处理

---

## 核心逻辑

### 分组目标

1. 根据学科代码形成粗分桶（代码前4-6位）
2. 在桶内基于项目关键词计算项目间相似度
3. 使用关键词相似度进行层次聚类，形成细粒度分组
4. 根据容量约束对过小组和过大组进行修正
5. 最终输出可解释、可回放的分组结果

### 关键词的使用

项目关键词（`gjc` 字段）是分组的核心依据：
- 解析项目的完整关键词列表（分号/逗号分隔）
- 计算两两项目的关键词重叠度（Jaccard 相似度或共现词数量）
- 关键词相似度高的项目优先聚在一组
- 不固定取前N个关键词，充分利用全部关键词信息

### 学科代码的作用

学科代码用于保证大领域边界：
- 代码前4位不同的项目，默认不放在同一组
- 在相同代码前缀内，再按关键词细分
- 字母代码与数字代码属于两套体系，不能混组

### 字母代码与数字代码隔离

字母开头学科代码与 7 位数字学科代码属于两套体系：
- 两套体系不能混在一个组里
- 同体系内可以通过关键词相似度判断拆分或合并

---

## 推荐流程

```
1. 获取项目列表（含学科代码、关键词字段）
2. 按学科代码前4位做粗分桶
3. 在每个桶内，解析项目关键词并计算项目间相似度矩阵
4. 基于关键词相似度进行层次聚类
5. 对超过 max_per_group 的组继续按关键词细分
6. 对小于 min_per_group 的组逐项目重分配
7. 按需生成组标题、理由和复核标记
8. 做字母代码 / 数字代码体系隔离检查
9. 输出分组结果 + 理由 + 复核标记
```

## 详细策略（代码+关键词联合分组）

### 1. 关键词提取

从 `sb_jbxx.gjc` 字段读取项目关键词：
- 按分号或逗号分隔解析关键词列表
- 清洗关键词（去除空格、统一大小写）
- 保留完整关键词集合，不限数量

示例：`"肿瘤;外泌体;分离分析;生物膜"` → `{"肿瘤", "外泌体", "分离分析", "生物膜"}`

### 2. 学科粗分

分组第一步利用学科代码形成粗粒度桶：
- 优先区分字母代码体系与数字代码体系
- 在同一体系内按代码前4位组织项目
- 代码前4位不同的项目默认不放在同一组

### 3. 关键词相似度计算

在每个代码桶内，计算项目间的关键词相似度：
- **Jaccard 相似度**：`|A ∩ B| / |A ∪ B|`
- **加权共现**：考虑关键词在组内的出现频率
- 相似度高的项目优先聚在一组

### 4. 层次聚类

基于关键词相似度进行层次聚类：
- 初始状态：每个项目为一个簇
- 迭代合并：相似度最高的两个簇合并
- 停止条件：达到目标组数或组大小约束
- 最终形成细粒度分组

### 5. 同代码拆分

同一学科代码下，关键词差异大的项目应分到不同组：

示例（代码 B0510）：
- 项目A关键词：`肿瘤, 外泌体, 分离` → 分到"生物医学分析组"
- 项目B关键词：`农药, 残留, 果蔬` → 分到"农产品安全组"
- 项目C关键词：`水污染, SERS, 检测` → 分到"环境检测组"

### 6. 小组与大组处理

对于小于 `min_per_group` 的组，不能只找一个目标组整体迁移，而要逐项目处理：
- 每个项目单独找最合适的组
- 一个小组里的项目可以分散到多个组
- 如果某个项目找不到合适目标组，保留待复核

对于超过 `max_per_group` 的组：
- 优先按语义切分成多个子主题
- 组标题优先采用规则生成，必要时再由 LLM 命名
- 每个子主题尽量保持内部一致

### 7. 体系隔离检查

分组结束后再做一次硬检查：
- 字母代码组里不能混入数字代码项目
- 数字代码组里不能混入字母代码项目
- 如已混入，则回退该轮合并结果并重新分配

---

## 结果要求

每个组至少输出：
- `group_name`
- `group_reason`
- `count`
- `needs_review`
- 组内每个项目的 `project_reason`
- 每个项目的 `confidence`

最终结果要能支持：
- 人工查看
- 追溯为什么这样分
- 重新运行时结果稳定

## 伪代码（代码+关键词联合分组）

```python
projects = load_fixed_grouping_test_projects()

# 1. 按学科代码前4位粗分
buckets = group_by_code_prefix(projects, prefix_length=4)

clusters = []
for bucket in buckets:
    # 2. 提取关键词
    keywords = extract_keywords(bucket)
    
    # 3. 计算关键词相似度矩阵
    similarity_matrix = compute_jaccard_similarity(keywords)
    
    # 4. 层次聚类
    sub_clusters = hierarchical_cluster(bucket, similarity_matrix, max_per_group)
    clusters.extend(sub_clusters)

# 5. 处理过小/过大组
clusters = redistribute_small_groups(clusters, min_per_group)
clusters = split_large_groups_by_keywords(clusters, max_per_group)

# 6. 体系隔离检查
clusters = enforce_code_system_separation(clusters)

return export_result(clusters)
```

## 约束参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_per_group` | 5 | 小组下限，低于则继续归并 |
| `max_per_group` | 15 | 每组上限，超过则拆分 |
| `code_prefix_length` | 4 | 学科代码前缀长度（粗分桶） |
| `similarity_threshold` | 0.3 | 关键词相似度阈值（低于不聚类） |
| `top_k_candidates` | 3 | 每个项目保留的候选组数 |
| `confidence_threshold` | 0.65 | 低于该值标记复核 |
| `enable_embedding` | false | 是否启用 embedding（当前用关键词替代） |
| `enable_llm` | false | 是否启用 LLM（当前用规则生成标题） |

---

## 运行建议

- **核心依据**：学科代码（前4位）+ 项目关键词（`gjc` 字段）
- 先按代码粗分，再在桶内按关键词相似度聚类
- 充分利用数据库现成关键词，避免从文本提取
- 关键词相似度用 Jaccard 或共现词计算，简单高效
- 组标题优先用学科名称+高频关键词规则生成
- 所有低置信度项目都要保留复核信息
