# 🔍 正文评审服务概述

## 服务定位

正文评审服务是科技管理系统的核心服务之一，负责对申报书正文进行多维度智能评审，输出评分、评审意见和问题清单。

该服务独立于形式审查、查重、分组等现有服务，专注于**内容质量评审**，为项目评审提供智能化决策支持。

## 与其他服务的关系

### 服务依赖

| 依赖服务 | 依赖方式 | 说明 |
|----------|----------|------|
| Common Layer | 直接依赖 | LLM、文件处理、数据库访问 |
| 形式审查 | 可选前置 | 可在形式审查通过后执行 |
| 查重 | 可选前置 | 可在查重通过后执行 |

## 核心能力

### 1. 多维度评审

覆盖项目质量的 9 个核心维度：

| 类别 | 维度 | 说明 |
|------|------|------|
| **核心维度** | 技术可行性 | 技术路线、技术成熟度、实施条件 |
| | 创新性 | 创新点、技术水平、创新程度 |
| | 团队能力 | 负责人资质、团队结构、研究基础 |
| **成果维度** | 预期成果 | 成果形式、量化指标、转化潜力 |
| | 社会效益 | 社会影响、推广价值、公益性 |
| | 经济效益 | 经济回报、市场前景、产业化 |
| **管理维度** | 风险控制 | 风险识别、应对措施、预案 |
| | 进度合理性 | 时间安排、里程碑、可控性 |
| | 合规性 | 完整性、规范性、政策符合 |

### 2. 智能评分

- **十分制评分**：每个维度 1-10 分
- **权重可配置**：支持用户自定义权重体系
- **等级判定**：自动判定 A/B/C/D/E 五级

### 3. 可解释性

- 每个维度输出详细评审意见
- 列出发现的问题清单
- 标注项目亮点
- 生成修改建议

### 4. 灵活配置

- 支持选择部分维度评审
- 支持自定义权重
- 支持指定评审章节

## 输入输出

### 输入

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| project_id | string | 是 | 项目ID |
| dimensions | array | 否 | 指定评审维度，默认全部 9 个 |
| weights | object | 否 | 自定义权重，默认使用预设权重 |
| include_sections | array | 否 | 指定评审章节 |

### 输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| project_id | string | 项目ID |
| project_name | string | 项目名称 |
| overall_score | float | 总分（加权平均，1-10分） |
| grade | string | 等级（A/B/C/D/E） |
| dimension_scores | array | 各维度评分详情 |
| summary | string | 综合评审意见 |
| recommendations | array | 修改建议列表 |
| created_at | datetime | 评审时间 |

## 技术栈

| 技术 | 用途 |
|------|------|
| **FastAPI** | API 框架 |
| **LangChain** | LLM 编排 |
| **GPT-4o / Claude 3.5** | 语义分析、评审意见生成 |
| **PyMuPDF** | PDF 解析 |
| **python-docx** | DOCX 解析 |
| **Pydantic** | 数据验证 |

## 服务路由

```
/api/v1/evaluation              # 单项目评审
/api/v1/evaluation/batch        # 批量评审
/api/v1/evaluation/{id}         # 获取评审结果
/api/v1/evaluation/dimensions   # 获取维度列表
/api/v1/evaluation/weights      # 获取权重模板
```
