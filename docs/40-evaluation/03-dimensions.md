## 画像自适应原则

九维框架固定，但不同类型项目的“看法”和“缺失项容忍度”不同。当前按 `project_profile` 做最小自适应，后续逐步过渡到 `Rubric Manager`：

| 项目画像 | 典型特征 | 当前放宽维度 | 放宽规则 |
|----------|----------|--------------|----------|
| `tech_rnd` | 技术研发、算法、模型、关键机理 | 暂无额外放宽 | 仍按标准研发口径评审 |
| `platform` | 平台建设、资源平台、数据库、公共服务 | `feasibility / schedule / risk_control / outcome / social_benefit / economic_benefit` | 可接受“建设目标/实施内容/核心建设内容/主要指标、效益”等替代章节 |
| `science_popularization` | 科普、宣教、活动、资源库、传播 | `feasibility / schedule / risk_control / outcome / social_benefit / economic_benefit` | 不强制要求独立“技术路线/进度安排/风险分析/社会效益/经济效益”章节，可按实施与推广类章节评估 |
| `demonstration` | 示范应用、推广、产业化、中试 | 预留 | 当前仅做画像识别，后续扩展专门口径 |
| `generic` | 证据不足 | 无 | 回退到默认通用口径 |

说明：

- 画像识别只基于正文规则，不依赖 LLM
- 若画像证据不足，默认按 `generic` 处理，避免误判
- 自适应的目标是“避免机械扣分”，不是“自动抬分”
- 后续 `Rubric Manager` 将在当前画像结果上进一步输出：
  - 必要维度 / 可选维度
  - 缺失项是降权、扣分还是允许替代
  - 每个维度的最低证据要求
  - 总评时的解释口径

## Rubric 优先原则

后续维度判断遵循以下原则：

1. 先按项目画像选择评审口径，再执行九维检查
2. 先构建维度级 `evidence pack`，再进入维度打分
3. 证据不足时允许保守评价，不强行给出确定性高分/低分
4. 不同类型项目的替代章节可以进入同一维度评估，但必须保留证据来源

## 核心维度

### 1. 技术可行性 (feasibility)

**评审目标**：评估技术路线的合理性、技术成熟度和实施可行性

**默认权重**：15%

**检查项配置**：

| 检查项 | 权重 | 评分标准 |
|--------|------|----------|
| 技术路线清晰度 | 30% | 路线图是否清晰、步骤是否明确 |
| 技术成熟度 | 30% | 核心技术是否成熟、是否有验证 |
| 实施条件完备性 | 20% | 设备、人员、资金是否到位 |
| 技术风险控制 | 20% | 是否识别技术风险、有应对方案 |

**评分标准**：

| 分数段 | 标准 |
|--------|------|
| 9-10 分 | 技术路线清晰完整，核心技术成熟且有验证，实施条件完备 |
| 7-8 分 | 技术路线较清晰，核心技术较成熟，实施条件基本具备 |
| 5-6 分 | 技术路线基本可行，技术成熟度一般，实施条件有待完善 |
| 3-4 分 | 技术路线不够清晰，技术成熟度较低，实施条件不足 |
| 1-2 分 | 技术路线模糊或不可行，技术不成熟，缺乏实施条件 |

**数据来源**：
- 技术方案章节
- 实施方案章节
- 技术路线图

**LLM Prompt 模板**：

请从以下方面评分（每项1-10分）：
1. 技术路线清晰度：技术路线图是否清晰、步骤是否明确
2. 技术成熟度：核心技术是否成熟、是否有验证

**数据来源**：
- 全文
- 申报要求
- 政策文件

## 等级判定

### 判定标准

| 等级 | 分数范围 | 描述 |
|------|----------|------|
| A | ≥ 9.0 | 优秀：项目质量优秀，建议优先支持 |
| B | ≥ 8.0 | 良好：项目质量良好，建议支持 |
| C | ≥ 6.0 | 中等：项目质量一般，需完善后考虑 |
| D | ≥ 4.0 | 较差：项目存在较多问题，建议修改后重新申报 |
| E | < 4.0 | 不合格：项目质量不合格，建议不予支持 |

### 判定逻辑

```python
def determine_grade(score: float) -> str:
    """根据分数判定等级"""
    if score >= 9.0:
        return "A"  # 优秀
    elif score >= 8.0:
        return "B"  # 良好
    elif score >= 6.0:
        return "C"  # 中等
    elif score >= 4.0:
        return "D"  # 较差
    else:
        return "E"  # 不合格
```
