## 分层架构

### Layer 1: API 层

- **职责**：路由定义、请求验证、响应格式化
- **组件**：`src/app/routes/evaluation.py`
- **特点**：
  - 自动生成 OpenAPI 文档
  - 请求参数验证
  - 统一错误处理

### Layer 2: Agent 层

- **职责**：流程编排、任务调度、结果聚合
- **组件**：`src/services/evaluation/agent.py`
- **特点**：
  - 异步并发执行
  - 流程可配置
  - 结果可追溯

### Layer 3: 检查器层

- **职责**：各维度评审检查逻辑
- **组件**：`src/services/evaluation/checkers/`
- **特点**：
  - 9 个独立检查器
  - 规则 + LLM 混合检查
  - 可独立测试

### Layer 4: 评分器层

- **职责**：维度评分、权重计算、等级判定
- **组件**：`src/services/evaluation/scorers/`
- **特点**：
  - 支持自定义权重
  - 多种评分模板
  - 等级自动判定

### Layer 5: 解析器层

- **职责**：文档解析、章节提取、字段定位
- **组件**：`src/services/evaluation/parsers/`
- **特点**：
  - 支持 PDF/DOCX
  - 智能章节识别
  - 关键字段定位

### Layer 6: 通用层

- **职责**：LLM 调用、文件处理、数据库访问
- **组件**：`src/common/`
- **特点**：
  - 已有组件复用
  - 统一接口

## 错误处理

### 错误类型

| 错误类型 | HTTP 状态码 | 说明 |
|----------|-------------|------|
| PROJECT_NOT_FOUND | 404 | 项目不存在 |
| DOCUMENT_PARSE_ERROR | 422 | 文档解析失败 |
| LLM_ERROR | 503 | LLM 调用失败 |
| INVALID_WEIGHTS | 400 | 权重配置无效 |
| INTERNAL_ERROR | 500 | 内部错误 |

### 错误响应格式

```json
{
  "error": {
    "code": "PROJECT_NOT_FOUND",
    "message": "项目不存在",
    "details": {
      "project_id": "202520014"
    }
  }
}
```

## 扩展性设计

### 新增评审维度

1. 在 `checkers/` 下创建新的检查器
2. 继承 `BaseChecker` 基类
3. 实现 `check()` 方法
4. 注册到 `CHECKER_REGISTRY`
5. 更新权重配置

### 新增权重模板

1. 在 `WeightCalculator.WEIGHT_TEMPLATES` 添加模板
2. 通过 API 暴露新模板

### 替换 LLM

- 使用 `common/llm` 统一封装
- 通过环境变量切换模型
- 无需修改业务代码

## 性能考虑

### LLM 调用优化

- **并发控制**：使用 Semaphore 限制并发数
- **结果缓存**：相同输入缓存 LLM 结果
- **超时控制**：设置合理的超时时间

### 文档解析优化

- **懒加载**：按需解析文档
- **增量解析**：只解析变更部分
- **结果缓存**：缓存解析结果

### 内存管理

- **大文件处理**：流式处理大文件
- **及时释放**：处理完成后释放内存
- **限制文档大小**：设置文档大小上限
