# 趋势预判分析链路

## 一、总链路

趋势预判建议采用如下链路：

```text
原始数据
  -> 主题映射与口径统一
  -> 时间窗聚合
  -> 结构变化检测
  -> 风险特征提取
  -> 预测与解释
  -> 结果标准化输出
```

## 二、阶段拆分

### 1. 口径统一

先统一主题口径、时间字段、项目状态和产出口径，避免后续把统计噪声误识别为趋势变化。

### 2. 结构变化检测

这一层用于发现：

- 主题升温 / 衰减
- 热点迁移
- 人才重心移动
- 协作结构变化

### 3. 风险特征提取

这一层不直接输出“结论文本”，而是产出可复用的结构化特征，例如：

- 高增长低转化
- 人才断层
- 协作弱化
- 高热度零产出

### 4. 预测与解释

预测层需要输出两个结果：

1. 未来窗口的趋势预测
2. 形成该预测的解释路径

## 三、方法分层

建议将方法拆成三层，而不是只用一层规则：

1. `detection`
   负责识别变化和异常
2. `forecasting`
   负责预测未来窗口
3. `explanation`
   负责把关键驱动因子结构化表达

## 四、当前原型可复用部分

旧原型中可迁入趋势预判的部分包括：

- 热点迁移图算法
- 宏观风险规则
- 图谱结构信号提取

但这些内容都应成为 `trend` 的子模块，而不是继续以 `step2 / step3` 命名存在。
