# 📊 数据接入概述

## 数据来源

奖励评审系统数据库，用于：
- **测试阶段**：验证功能正确性
- **训练阶段**：优化 AI 模型效果
- **交付阶段**：提供 API 服务（不含原始数据）

## 接入原则

| 原则 | 说明 |
|------|------|
| **只读** | 禁止任何写入操作，保护甲方数据 |
| **按需加载** | 根据服务需求加载对应数据，不一次性接入全部 |
| **只读** | 开发调试直连甲方服务器，无本地副本 |
| **可扩展** | 支持后续接入更多数据库（如项目评审） |

## 分组测试数据约定

为保证分组功能联调、回归和效果对比的一致性，后续默认使用**项目评审库中的固定业务子集**作为测试数据来源。

约定如下：

- 数据主体来自项目申报基础信息与申报状态信息的关联结果
- 仅选取**同一业务批次**下、**已通过公开审核/业务审核**的项目
- 该数据集仅用于分组功能测试、参数调优和结果对比
- 文档、日志、示例返回中不暴露真实业务批次标识、单位信息、联系人信息等敏感字段

在实现层面，可将其理解为：

- 主表：`Sb_Jbxx`
- 状态表：`Sb_Sbzt`
- 过滤条件：某固定业务批次 + 审核通过

> ⚠️ 注意：具体批次标识、原始 SQL、真实主键值仅保留在受控环境中，不写入公开文档。
