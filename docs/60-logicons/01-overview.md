# 🧠 逻辑自洽校验服务概述

## 服务定位

逻辑自洽校验服务（LogiCons）用于识别申报书/任务书中的跨段落逻辑矛盾与账目不平问题。系统不再只检查单段文本，而是将整份文档作为整体进行结构化语义解析，自动关联人、物、财、时、指标等实体，构建临时文档知识图谱，并通过 AI Agent 执行全局冲突比对。

### 核心能力

1. **跨章节时间一致性校验**：识别执行期、里程碑、任务进度之间的时间跨度冲突。
2. **跨章节金额一致性校验**：识别总额、分项、年度预算之间的数学不一致。
3. **实体关系一致性校验**：识别项目名称、承担单位、负责人、考核指标的语义冲突。
4. **可定位冲突证据**：输出冲突项、来源章节、证据片段、修正建议。

### 解决的核心问题

| 现状痛点 | 解决方案 |
|----------|----------|
| 执行期填 2 年，进度表却跨 3-4 年 | 提取时间实体并统一归一化后做区间冲突比对 |
| 资金总额 50 万，明细求和到 70-100 万 | 提取金额字段并做公式化校验与容差判定 |
| 章节多、术语混用，人工核对易漏看 | 构建文档临时知识图谱 + Agent 全局推理 |
| 审核意见无法定位到具体文本 | 返回冲突证据（章节、页码、原文片段） |

## 业务流程

```
┌──────────────────────────────────────────────────────────────────┐
│                    逻辑自洽校验全流程                              │
└──────────────────────────────────────────────────────────────────┘

      用户上传申报书/任务书 (DOCX/PDF)
                  │
                  ▼
┌──────────────────────────┐
│ 1. 文档解析与分段切片      │
│ - 章节识别                │
│ - 表格识别                │
│ - 页码/段落索引           │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│ 2. 实体抽取与归一化        │
│ - 人/单位/项目            │
│ - 时间/金额/指标          │
│ - 同义词与别名归并         │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│ 3. 临时知识图谱构建        │
│ - 实体节点                │
│ - 关系边 (约束/隶属/引用) │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│ 4. Agent 冲突比对         │
│ - 数值一致性              │
│ - 时间一致性              │
│ - 语义一致性              │
└───────────┬──────────────┘
            │
            ▼
┌──────────────────────────┐
│ 5. 冲突定位与报告生成      │
│ - 冲突等级                │
│ - 证据片段                │
│ - 修正建议                │
└──────────────────────────┘
```

## 模块结构

```
src/
├── services/
│   └── logicons/                      # 逻辑自洽服务 (60-logicons)
│       ├── service.py                 # 服务入口
│       ├── config.py                  # 参数配置
│       ├── parser/
│       │   ├── section_parser.py      # 章节/段落解析
│       │   └── table_parser.py        # 表格解析与结构化
│       ├── extractor/
│       │   ├── entity_extractor.py    # 实体提取
│       │   └── normalizer.py          # 归一化（金额/时间/别名）
│       ├── graph/
│       │   ├── builder.py             # 临时知识图谱构建
│       │   └── schema.py              # 图谱 Schema
│       ├── rules/
│       │   ├── timeline_rule.py       # 时间一致性规则
│       │   ├── budget_rule.py         # 资金一致性规则
│       │   └── indicator_rule.py      # 指标一致性规则
│       ├── agent/
│       │   └── consistency_agent.py   # 全局冲突推理 Agent
│       └── api.py                     # HTTP API
│
├── common/
│   ├── file_handler/                  # 复用文档解析能力
│   ├── llm/                           # 复用 LLM 统一封装
│   └── models/
│       └── logicons.py                # 数据模型
```

## 技术选型

| 技术 | 用途 |
|------|------|
| **LangChain LCEL** | Agent 编排、链式推理 |
| **Pydantic** | 结构化实体与校验结果模型 |
| **规则引擎（自定义）** | 数值/时间/语义一致性规则执行 |
| **Docx/PDF 解析组件** | 段落、表格、页码信息提取 |
| **LLM + Pattern 混合抽取** | 复杂语义抽取与可解释校验 |

## 上下游依赖

### 上游输入

| 来源 | 内容 |
|------|------|
| 用户上传文件 | 申报书、任务书（DOCX/PDF） |
| 调用参数 | 阈值配置、校验范围、输出模式 |

### 下游输出

| 去向 | 内容 |
|------|------|
| 申报端 | 冲突提示与修正建议 |
| 审核端 | 高风险冲突清单与证据定位 |
| 归档系统 | 结构化校验结果 JSON |

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/logicons/check` | POST | 提交文档并执行全局逻辑一致性校验 |
| `/api/v1/logicons/{check_id}` | GET | 查询校验结果 |
| `/api/v1/logicons/rules` | GET | 查询可用规则与阈值 |
