## 层次说明

### 1. API Layer（API 层）

- **框架**: FastAPI + LangServe
- **职责**: 接收 HTTP 请求，返回响应
- **特点**: 
  - 自动生成 OpenAPI 文档
  - 请求验证
  - 统一错误处理

### 2. Service Layer（服务层）

- **职责**: 业务逻辑处理
- **核心组件**:
  - **Agent**: 智能决策单元，协调规则和 LLM
  - **规则引擎**: 基于预定义规则的检查
  - **文档解析器**: 版式分析、内容提取
- **特点**: 
  - 每个服务独立
  - 复用 Common Layer

### 3. Common Layer（通用组件层）

- **职责**: 提供跨服务共享的能力
- **模块**:
  - **models**: 通用数据模型
  - **llm**: LLM 统一封装
  - **file_handler**: 文件处理（PDF、图片、OCR）
  - **vision**: 视觉能力（目标检测、图像分析）
  - **tools**: 工具函数
- **特点**:
  - 高内聚、低耦合
  - 可复用、可替换

## 核心技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **API** | FastAPI | 高性能异步框架 |
| **LLM 编排** | LangChain LCEL | 统一Runnable接口 |
| **多模态** | GPT-4V / Claude 3.5 / Qwen-VL | 图像理解 |
| **目标检测** | YOLO / GroundingDino | 物体检测 |
| **OCR** | PaddleOCR | 文字识别 |
| **文档解析** | PyMuPDF / LayoutParser | 版式分析 |

## 数据流

### 形式审查流程

```
1. 用户上传文件
       │
       ▼
2. API 接收 → 存储临时文件
       │
       ▼
3. Service 层调用 ReviewAgent
       │
       ├─→ 文档分类（LLM）
       │
       ├─→ 版式解析（目标检测 + OCR）
       │
       ├─→ 规则检查（签名/盖章/前置条件）
       │
       └─→ LLM 补充判断（复杂版式）
       │
       ▼
4. 结果聚合 → 返回审查结果
```

## 扩展性设计

### 新增服务

```
1. 在 services/ 下创建新目录
2. 继承 BaseService
3. 实现业务逻辑（可复用 Common Layer）
4. 在 app/routes/ 添加路由
```

### 新增规则

```
1. 在 services/review/rules/checkers/ 下创建检查器
2. 继承 BaseRule
3. 实现 check() 方法
4. 注册到 ReviewAgent
```

### 替换 LLM

```
1. 在 common/llm/ 下实现新 Client
2. 继承 BaseLLM
3. 配置切换（环境变量）
```
