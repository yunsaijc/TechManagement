# LogicOn 文档解析方案

## 解析目标

将单份申报书或任务书解析为“可定位、可抽取、可比对”的中间表示，至少包括：

- `raw_text`：全文文本
- `sections[]`：章节/主题块（含标题、内容、DocSpan）
- `table_rows[]`：表格行文本（用于预算/指标等数值抽取）

## 输入类型

- PDF：可能为可复制文本或扫描件
- DOCX：通常结构更规范（标题层级/表格更完整）
- 纯文本：用于调试与快速接入

## 解析策略

### 1) 文件解析

复用 `src/common/file_handler`：

- PDF：抽取文字与页码，必要时使用 OCR（若已集成）
- DOCX：抽取段落、表格文本，尽量保留标题层级信息

输出统一为文本块序列：

- `chunk_text`
- `page`（若可得）
- `source`（paragraph/table/ocr）

### 2) 章节/主题切分

目标：在“全文”中定位关键章节的证据边界，避免抽取时引入无关上下文。

关键章节（示例关键词）：

- 基本信息/起止年月/执行期
- 进度安排/阶段目标/里程碑
- 预算表/经费预算/资金来源/预算明细
- 绩效目标表/考核指标/验收指标
- 人员及分工/项目组成员

切分方式：

- DOCX：优先使用标题段落作为 section title；title 下连续内容归入同一 section
- PDF：基于标题关键词与行间距/换行密度启发式切分；结合“表格标题/表格表头/表格行”标记增强切段

输出：

- `sections[] = [{title, content, span}]`

其中 `span` 为 DocSpan，至少携带 `section_title` 与 `snippet`；如果能定位到页码则写入 `page`。

## 结构化字段抽取

### 1) 时间抽取（执行期与进度节点）

抽取目标：

- 执行期：起止年月/总年限（如 2 年、24 个月）
- 进度节点：里程碑日期（年月/季度）、年度集合（第一年/第二年/2026 年等）

抽取来源：

- 执行期：基本信息表、项目起止年月、签订年度等位置
- 进度节点：进度安排和阶段目标/详细任务进度安排章节

归一化：

- 起止年月 → `start_year_month` / `end_year_month`
- 年限 → `duration_months`
- 里程碑日期集合 → `milestone_year_months[]`
- 年份集合 → `years[]`

### 2) 预算抽取（总额/科目/单位预算）

抽取目标：

- 总预算/资金申请总额/下达总额
- 科目明细（设备费/材料费/劳务费/业务费等）
- 单位预算明细（承担单位/合作单位）

实现建议：

- 复用 perfcheck 的表格行抽取与纠偏逻辑（优先表格行而非 LLM）
- 对金额单位做归一化（元/万元），并记录原单位用于展示

### 3) 指标抽取（目标值/单位/约束）

抽取目标：

- 指标名（三级细项指标）
- 指标值、单位、约束（≥/≤/=/区间）
- 来源章节（绩效目标表/验收指标等）

实现建议：

- 复用 perfcheck 的“绩效目标表抽取 + 补齐实施期目标”逻辑作为基础
- 额外从“验收考核指标/阶段目标”中抽取并打上 `scope`，用于后续冲突比对时区分“过程指标”与“最终指标”

## 证据定位（DocSpan）

DocSpan 的目标是让冲突结果可复核：

- 优先使用页码（PDF 可得）
- 同时保留 `section_title` 与 `snippet`

当无法提供精确字符区间时：

- 仍返回 `section_title + snippet`，保证人工可快速定位

## 不确定性与可控边界

- 扫描件 PDF：若 OCR 质量不佳，时间/金额抽取可能不全，规则应输出 YELLOW 而非 RED
- 同义指标：需要语义归一化，建议在 `enable_llm=true` 时提升召回，并保留证据与置信度
