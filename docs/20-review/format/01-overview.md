# 🔍 附件级形式审查概述

## 服务定位

附件级形式审查是形式审查体系的底层能力，负责对单个附件进行检查，包括签字、盖章、完整性和少量单附件一致性检查。

它不直接承担完整的项目级形式审查职责。项目级材料齐套性、项目类型差异、跨附件判断和外部校验应由上层项目级形式审查负责。

## 功能特性

### 1. 多模态智能审查

- **签字检测**: 自动识别文档中的签名区域，验证签字是否存在
- **盖章检测**: 识别印章、戳记，验证盖章是否符合要求
- **版式分析**: 分析文档版式，处理复杂排版

### 2. 规则 + AI 混合

- **规则引擎**: 基于预定义规则的精确检查
- **LLM 补充**: 多模态大模型处理复杂版式和语义理解
- **协同决策**: Agent 协调规则和 LLM，取长补短

### 3. 适用范围

| 附件类型示例 | 检查项 |
|----------|--------|
| 专利证书 | 签字、盖章、专利号一致性 |
| 验收报告 | 前置条件、签字、盖章 |
| 检索报告 | 检索结果完整性 |
| 奖励证书 | 证书一致性 |
| 主要完成人情况表 | 签字、盖章、工作单位一致性 |

奖励平台 SMB 单文件主线当前规划覆盖以下 `doc_type` 字典类型：

| `doc_type` | 中文名称 | 注释 | 主要校验 |
|------|------|------|------|
| `tjdwyj` | 提名单位意见表 (`10.1`) | 奖励项目提名单位出具的意见页 | 盖章单位与推荐单位全称一致 |
| `gzdwyj` | 候选人工作单位意见 (`10.2`) | 候选人所在工作单位出具的意见页 | 盖章单位与候选人工作单位一致 |
| `wcr` | 主要完成人情况表 (`10.3`) | 境内主要完成人签字盖章页 | 姓名、工作单位、完成单位、签字、盖章一致性 |
| `wjwcr` | 外籍主要完成人情况表 (`10.3`) | 外籍主要完成人签字盖章页 | 姓名、工作单位、完成单位、签字、盖章一致性 |
| `wcdw` | 主要完成单位情况表 (`10.4`) | 完成单位签章页 | 单位名称、法定代表人、盖章一致性 |
| `hzdw` | 河北省内主要合作单位情况表 (`10.5`) | 合作单位签章页 | 单位名称、盖章一致性 |

## 业务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                       格式审查流程                                │
└─────────────────────────────────────────────────────────────────┘

     用户上传单个附件
           │
           ▼
    ┌─────────────┐
    │  文件接收   │  ← 验证格式、大小、类型
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ 附件识别   │  ← 由调用方指定类型；奖励平台路径接口使用 doc_type 字典值
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │  版式解析   │  ← 提取文字、图像区域
    └──────┬──────┘
           │
           ▼
    ┌─────────────────────────────────────┐
    │          Agent 决策                  │
    │  ┌─────────┐    ┌─────────────┐   │
    │  │ 规则检查 │ +  │   LLM 分析   │   │
    │  └────┬────┘    └──────┬──────┘   │
    │       └─────────────────┘          │
    └──────────────┬────────────────────┘
                   │
                   ▼
    ┌─────────────────────────────┐
    │        结果聚合             │
    │  - 检查项列表              │
    │  - 单附件总结             │
    └──────────────┬────────────┘
                   │
                   ▼
           附件审查结果输出
```

## 模块结构

```
src/
├── common/                         # 通用层
│   ├── vision/                    # 底层视觉能力
│   │   ├── detector.py            # YOLO 检测器
│   │   └── multimodal.py          # 多模态 LLM
│   └── extractors/               # 统一提取器（Layer 4）
│       ├── signature.py            # 签字提取器
│       ├── stamp.py               # 盖章提取器
│       └── field.py               # 字段提取器
│
└── services/review/               # 形式审查服务
    ├── agent.py                   # 附件级审查 Agent (Layer 6)
    ├── extractor.py               # 内容提取（OCR + 文字解析）
    ├── parser.py                  # 文档解析
    └── rules/                     # 规则引擎 (Layer 5)
        ├── base.py                # 规则基类
        ├── registry.py            # 规则注册表
        └── checkers/              # 具体检查器
            ├── signature.py       # 签字检查
            ├── stamp.py          # 盖章检查
            ├── prerequisite.py   # 前置条件
            └── work_unit.py      # 工作单位一致性检查
```

## 核心技术

| 技术 | 用途 |
|------|------|
| **LangChain LCEL** | Agent 编排、链式调用 |
| **多模态 LLM** | 复杂版式分析、语义理解 |

## 与项目级形式审查的关系

附件级形式审查负责生成单附件证据，供上层项目级形式审查消费。

典型输出包括：

- 是否有签字
- 是否有盖章
- 是否完整
- 是否提取到关键字段
- 单附件是否通过

项目级材料齐套性、执行期限制、资金比例、合作单位要求、失信和重复申报等规则，不属于本层职责。

奖励平台单文件签章校验在本层额外承担：

- 按 `project_id + file_path + doc_type` 定位奖励附件
- 通过 SMB 读取远端 PDF
- 对签字盖章页做方向纠正与图像增强
- 使用 `tjdwyj/gzdwyj/wcr/wjwcr/wcdw/hzdw` 这套 `doc_type` 字典值
- 输出结构化识别结果：
  - 是否有签字
  - 是否有盖章
  - 识别的签字内容
  - 识别的印章内容
