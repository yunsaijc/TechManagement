# 🤖 Agent 设计

## 概述

查重服务采用“候选召回 + 多层过滤 + 人工复核”的分层架构。核心目标不是把所有命中都输出，而是尽量输出**连续性强、可解释、可复核**的有效重复片段。

## 核心设计原则

1. **语义分句**：按句末标点切分，保留原始位置映射
2. **多层过滤**：在比对前后都过滤模板、标题、短句和表格噪声
3. **N-gram 召回**：使用滑动窗口生成候选片段，优先保证召回
4. **Winnowing 合并**：将连续命中的片段合并为更完整的重复区间
5. **pair 级判定**：最终只保留足够完整、足够可信的文档对

## 查重策略

业界成熟系统通常采用“**先找候选，再做过滤，最后人工复核**”的模式。

- **候选层**：尽量多找可能相似的片段
- **过滤层**：去掉模板句、标题、短句、表格噪声和低信息密度片段
- **结果层**：只输出连续性足够强的有效重复
- **解释层**：记录每个 pair 的保留和过滤原因

## 关键组件

### 1. SentenceTokenizer

负责按句末标点切分，并保留 `start_pos` / `end_pos` / 行号映射，便于后续回溯。

### 2. TemplateFilter

负责识别模板、标题、表格和过短片段，用于前置过滤和后置过滤。

### 3. NGramSplitter

负责把句子切成固定长度的 N-gram，为候选召回提供基础特征。

### 4. ComparisonEngine

负责构建指纹索引、检测连续命中、合并区间，并抽取可读的源文本。

### 5. ResultAggregator

负责把引擎输出整理成最终结果，并拆分为有效片段、模板片段和过滤原因。

## pair 级过滤

为了减少假阳性，最终输出不应直接接受所有匹配结果，而应增加 pair 级规则：

- **最小连续长度**：过滤零散碎片
- **最小覆盖率**：避免只有很短的局部命中
- **最小有效字符数**：排除信息量过低的 pair
- **模板占比阈值**：模板内容占比过高时直接过滤
- **人工可读性**：保留能够明确解释的 pair

建议在 `aggregator.py` 中统一完成 pair 级判断，并在 debug 输出里记录 `filter_reason`。

## 推荐调参方向

| 参数 | 建议 | 目标 |
|------|------|------|
| `winnowing_window` | 偏大 | 降低碎片命中 |
| `min_match_length` | 提高 | 去掉短片段噪声 |
| `max_gap` | 适度控制 | 允许少量断句，但避免拼接过度 |
| `min_continuous_match` | 严格 | 保留连续命中 |
| `pair_quality_score` | 新增 | 提升输出接近人工判断 |
