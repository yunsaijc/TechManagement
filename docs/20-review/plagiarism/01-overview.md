# 🔍 查重服务概述

## 服务定位

查重服务是形式审查的子服务之一，负责对上传的文档进行文本相似度检测，识别重复申报、抄袭等情况。

## 功能特性

### 1. 多格式文本提取

- **PDF 解析**: 复用 `common/file_handler/pdf_parser.py`
- **DOCX 解析**: 复用 `common/file_handler/docx_parser.py`
- **纯文本提取**: 自动清理格式，保留核心内容

### 2. Section 区域提取

- **标题匹配**: 使用正则表达式匹配文档章节标题
- **可配置**: 支持自定义待查重区域
- **精确提取**: 只提取有价值的查重内容

### 3. 智能文本预处理

- **语义分句**: 按标点符号（。！？；）切分，保留句子边界和位置映射
- **前置过滤**: 比对前标记模板内容位置，避免干扰连续匹配
- **后置过滤**: 比对完成后对匹配结果二次过滤，区分模板重复和有效重复
- **白名单机制**: 常见模板短语加入白名单，不计入有效重复

### 4. Winnowing 算法比对

- **N-gram 切分**: 生成 8-gram 滑动窗口，减少碎片化
- **指纹索引**: 使用 MD5 指纹（取前8位）构建倒排索引
- **Winnowing 窗口**: 使用滑动窗口检测连续匹配，过滤分散的噪声匹配
- **连续区间合并**: 合并相邻的匹配区间，形成完整的重复片段

### 5. 批量查重

- **全量比对**: 所有文档两两比对
- **结果分类**: 高相似度 / 中相似度 / 低相似度
- **位置追溯**: 精确定位到行号、段落、Section

## 业务流程（业界最佳实践）

```
┌─────────────────────────────────────────────────────────────────┐
│                         查重流程                                 │
└─────────────────────────────────────────────────────────────────┘

     用户上传文件（多个）
           │
           ▼
    ┌─────────────────┐
    │   文件接收      │  ← 验证格式（PDF/DOCX）
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │   文本提取      │  ← PDF/DOCX 解析
    │  (file_handler) │    提取段落、表格、标题
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  句子级分词     │  ← 按标点分句（。！？；）
    │(SentenceTokenizer)│   保留位置映射
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  Section 提取   │  ← 仅主文档提取指定区域
    │(SectionExtractor)│   对比文档使用全文
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  前置模板过滤   │  ← 标记模板内容位置
    │(TemplatePreFilter)│  排除区间，避免干扰
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  N-gram 切分    │  ← 生成 8-gram
    │ (NGramSplitter)  │    不过滤，保留原始位置
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  指纹索引构建   │  ← MD5 指纹（前8位）
    │(ComparisonEngine)│  倒排索引
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  Winnowing 窗口  │  ← 滑动窗口检测
    │(ComparisonEngine)│  连续匹配
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  连续区间合并   │  ← 合并相邻区间
    │(ComparisonEngine)│  形成完整片段
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  后置模板过滤   │  ← 二次过滤
    │ (TemplateFilter) │  区分模板重复 vs 有效重复
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │   结果聚合      │  ← 片段合并
    │(ResultAggregator)│ 位置追溯
    └────────┬────────┘  分类输出
             │
             ▼
       查重结果输出
```

## Section 配置

系统支持通过配置文件定义待查重的文档区域。

### 默认配置

适用于项目申报书类文档，需要查重的区域如下：

| Section | 起始标题 | 结束标题 |
|---------|----------|----------|
| 项目立项背景及意义 | 项目立项背景及意义 | 项目简介 |
| 项目简介 | 项目简介 | 第一部分 |
| 第一部分 项目实施内容及目标 | 第一部分\s*项目实施内容及目标 | 第二部分 |
| 第二部分 申报单位及合作单位基础 | 第二部分\s*申报单位及合作单位基础 | 第三部分 |
| 第三部分 项目实施计划及保障措施 | 第三部分\s*项目实施计划及保障措施 | 文档结尾 |

### 配置定义

配置文件位于 `src/services/plagiarism/config.py`：

```python
PLAGIARISM_SECTION_CONFIG: Dict[str, Dict] = {
    "default": {
        "name": "默认配置",
        "sections": [
            {
                "name": "项目立项背景及意义",
                "start_pattern": r"项目立项背景及意义",
                "end_pattern": r"项目简介",
            },
            # ... 更多 section
        ]
    },
}
```

### 自定义配置

支持通过 API 参数传入自定义 section 配置（详见 API 文档）。

## 核心技术

| 技术 | 用途 |
|------|------|
| **PyMuPDF** | PDF 解析 |
| **python-docx** | DOCX 解析 |
| **正则表达式** | Section 区域匹配 |
| **语义分句** | 按标点符号分句，而非按行 |
| **N-gram 滑动窗口** | 文本切分 + 指纹生成 |
| **SimHash** | 指纹算法，加速大规模比对 |
| **多层过滤机制** | 白名单 + 标题检测 + 短句过滤 |

## 查重策略：后置过滤

业界最佳实践（知网、Turnitin）采用**先比对、后过滤**的策略：

1. **全文N-gram指纹比对**：位置精确对齐
2. **找出所有重复片段**
3. **后置过滤**：对匹配结果进行模板检测

### 为什么后置过滤？

- **位置对齐**：主文档和对比文档都在原始位置上进行比对
- **过滤准确**：基于匹配到的具体内容判断是否是模板
- **可区分性**：可区分"模板重复"和"有效重复"

### 输出结构

```json
{
  "total_similarity": 0.52,        // 总重复率
  "effective_similarity": 0.35,       // 有效重复率（过滤后）
  "template_segments": [...],          // 模板重复片段
  "effective_segments": [...]           // 有效重复片段
}
```

## 模板过滤规则

### 白名单模板短语

以下模板句式不计入重复：

```python
TEMPLATE_PHRASES = [
    r"为了认真贯彻落实.*?要求",
    r"根据.*?规定",
    r"特制定本.*?",
    r"本办法适用于",
    r"现将.*?情况汇报如下",
    r"\d+[万千百亿].*?元",  # 金额模板
]
```

### 标题行检测

以下模式视为标题，跳过比对：

```python
HEADING_PATTERNS = [
    r"^第[一二三四五六七八九十百]+部分",  # 第一部分
    r"^[一二三四五六七八九十]、",          # 一、二、三
    r"^\d+\.\d+",                         # 1.2.3
    r"^[A-Z]\.",                         # A. B. C.
    r"^\([一二三四五六七八九十]\)",        # (1) (一)
]
```

### 短句过滤

- 独立句子 < 15 字 → 跳过
- 段落内短句 → 保留（避免误杀）

### 表格相关

```python
TABLE_PATTERNS = [
    r"^\[表格行\d+\]",      # [表格行1]
    r"^\s*[\u4e00-\u9fa5]+\s*\|\s*[\u4e00-\u9fa5]+",  # "项目 | 金额"
]
```
