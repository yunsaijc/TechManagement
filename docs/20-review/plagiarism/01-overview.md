# 🔍 查重服务概述

## 服务定位

查重服务是形式审查的子服务之一，负责对上传的文档进行文本相似度检测，识别重复申报、抄袭和模板化拼接等问题。

## 设计目标

- **高召回**：尽量找出可疑重复片段
- **低误报**：避免模板句、固定表述和碎片化命中污染结果
- **可解释**：每个输出结果都能说明保留/过滤原因
- **可复核**：最终结果支持人工快速判断

## 功能特性

### 1. 多格式文本提取

- **PDF 解析**：复用 `common/file_handler/pdf_parser.py`
- **DOCX 解析**：复用 `common/file_handler/docx_parser.py`
- **纯文本提取**：自动清理格式，保留核心内容

### 2. Section 区域提取

- **标题匹配**：使用正则表达式匹配文档章节标题
- **可配置**：支持自定义待查重区域
- **精确提取**：只提取有价值的查重内容

### 3. 智能文本预处理

- **语义分句**：按标点符号（。！？；）切分，保留句子边界和位置映射
- **前置过滤**：在比对前标记模板内容位置，避免干扰连续匹配
- **后置过滤**：比对完成后对匹配结果二次过滤，区分模板重复和有效重复
- **白名单机制**：常见模板短语加入白名单，不计入有效重复

### 4. Winnowing 算法比对

- **N-gram 切分**：生成 8-gram 滑动窗口，保证局部召回能力
- **指纹索引**：使用稳定指纹构建倒排索引，便于快速候选召回
- **Winnowing 窗口**：先找连续命中，再做区间合并，减少碎片噪声
- **连续区间合并**：合并相邻的匹配区间，形成更完整的重复片段

### 5. 批量查重

- **全量比对**：所有文档两两比对
- **结果分类**：高相似度 / 中相似度 / 低相似度
- **位置追溯**：精确定位到行号、段落、Section

## 业界最佳实践对齐

成熟查重系统通常采用“**候选召回 + 多层过滤 + 人工复核**”模式，而不是“命中就输出”。

- **候选层**：负责找可能相似的片段，尽量保持高召回
- **过滤层**：排除模板句、目录句、引用句、短句噪声和低信息密度片段
- **结果层**：只输出连续性强、覆盖足够、可人工复核的 pair
- **解释层**：记录每个 pair 的保留/过滤原因，便于回看和调参

## 结果判定原则

成熟查重系统一般不会把“所有 pair”都直接输出，而是先通过 pair 级规则筛掉明显噪声：

- **最小有效长度**：过滤过短片段，避免碎片命中
- **最小连续覆盖**：要求命中片段连续，而不是零散拼接
- **最小覆盖率**：主文档和来源文档都要有足够覆盖
- **模板降权/剔除**：引用、申报说明、通用套话不计为有效重复
- **人工可解释**：每个输出 pair 都应能说明为何保留

## 推荐调参方向

基于当前测试样本，建议优先提高结果质量，而不是继续放宽阈值扩大召回。

| 参数 | 方向 | 目的 |
|------|------|------|
| `winnowing_window` | 保持偏大 | 降低碎片命中 |
| `min_match_length` | 提高 | 去掉短碎片噪声 |
| `max_gap` | 适度提高 | 允许少量断句，但避免过度拼接 |
| `min_continuous_match` | 保持严格 | 只保留连续命中 |
| `pair_quality_score` | 新增 | 让最终输出更接近人工判断 |

## 查重策略：后置过滤

业界最佳实践采用**先比对、后过滤**的策略：

1. **全文 N-gram 指纹比对**：位置精确对齐
2. **找出所有重复片段**
3. **后置过滤**：对匹配结果进行模板检测
4. **pair 级判定**：只保留连续性强、覆盖足够的有效重复
5. **输出**：总重复 + 有效重复（过滤后）

### 为什么后置过滤？

- **位置对齐**：主文档和对比文档都在原始位置上进行比对
- **过滤准确**：基于匹配到的具体内容判断是否是模板
- **可区分性**：可区分“模板重复”“候选命中”和“有效重复”
- **可解释性**：输出保留和过滤原因，方便人工复核

### 输出结构

```json
{
  "total_similarity": 0.52,
  "effective_similarity": 0.35,
  "candidate_segments": [],
  "filtered_pairs": [],
  "final_pairs": [],
  "template_segments": [],
  "effective_segments": []
}
```
