# 趋势预判对象与指标

## 一、定位

`Trend = Baseline World Forecast`

趋势预判不是“热点图展示”，也不是把过去几年的统计结果换个颜色再画出来。

它要回答的是：

- 现在的科研布局是什么样
- 过去几年发生了哪些结构变化
- 在当前政策 regime 不变时，未来 1-2 个年度周期大概率会如何延续
- 哪些变化已经开始积累风险，值得领导提前干预

因此，趋势体系必须先定义完整业务对象，再在其上构建统计层与预测层。

## 二、对象层次

趋势预判与沙盘推演共享同一套三层对象体系。

### 1. 实体层

实体层只承载真实业务对象，不承载结论。

核心实体如下：

| 实体 | 含义 | 当前主要来源 | 数据属性 |
|---|---|---|---|
| `Project` | 从申报到合同的项目生命周期主实体 | `Sb_Jbxx` 及其后续链路 | 观测 |
| `Topic` | 指南方向/专项方向等治理主题 | `Sb_Jbxx.zndm`、指南名称、图谱映射 | 观测+映射 |
| `Institution` | 承担单位、合作单位等组织主体 | 项目库主体字段、图谱机构节点 | 部分观测 |
| `Person` | 项目负责人、参与人员、图谱人物节点 | 项目库人员字段、图谱人物节点 | 部分观测 |
| `Review` | 评审事件及阶段结果 | `PS_XMPSXX` | 观测 |
| `Funding` | 申报经费、立项经费、合同经费等资金事件 | `Sb_Jfgs`、`Ht_Jfgs` | 观测 |
| `Portfolio` | 领导实际管理的全省科研组合结构 | 由下层聚合形成 | 派生 |

说明：

- `Project` 是证据颗粒，不是领导页面主视角。
- `Portfolio` 是领导决策对象，但它不是独立主数据表，而是由项目、主题、机构、人员的状态聚合形成。

### 2. 关系层

关系层承载结构演化，回答“谁和谁连在一起、结构如何变化”。

核心关系如下：

| 关系 | 含义 | 当前主要来源 | 数据属性 |
|---|---|---|---|
| `Project -> Topic` | 项目归属哪个主题 | 项目库字段映射、图谱映射 | 观测+映射 |
| `Project -> Institution` | 项目由哪些单位承担/协同 | 项目库、图谱 | 观测+部分补齐 |
| `Project -> Person` | 项目与负责人/参与人关系 | 项目库、图谱 | 部分观测 |
| `Topic <-> Topic` | 主题之间的相邻、迁移、共现 | 图谱与派生计算 | 代理 |
| `Institution <-> Institution` | 机构协作关系 | 图谱与派生计算 | 代理 |
| `Person <-> Topic` | 人才跨主题活跃和流动 | 图谱与派生计算 | 代理 |

### 3. 统计层

统计层是趋势计算的工作层，不是业务本体。

核心统计对象如下：

| 统计对象 | 粒度 | 作用 |
|---|---|---|
| `topic_year_state` | `topic × year` | 主题年度状态与短期趋势判断的主分析层 |
| `topic_institution_year_state` | `topic × institution × year` | 看机构集中度、主导格局、进入退出 |
| `topic_person_year_state` | `topic × person × year` | 看人才活跃度、断层风险、桥接性 |
| `project_cohort_outcome` | `application_cohort × topic` | 看同一申报批次后续走向 |
| `portfolio_year_state` | `portfolio × year` | 领导视角的全局结构汇总 |

原则：

- `topic × year` 是主分析层，但不是世界本体。
- 任何领导结论都必须能追回实体层和关系层证据。

### 4. 辅助信号层

外部信息可以存在，但只能作为辅助信号层，不能冒充主数据。

可接入的辅助信号包括：

- 政策文本信号
- 新闻/舆情信号
- 论文/专利热度
- 行业景气与产业事件

使用原则：

- 只参与解释、校验、排序加权
- 不直接改写项目事实、经费事实、主题事实
- 必须单独标注来源、时间、覆盖范围和可信度

## 三、指标体系

领导视角下，趋势指标应围绕四类问题组织：

1. 看现在
2. 看变化
3. 看未来
4. 看风险

### 1. 看现在：现状指标

用于描述当前科研布局和资源结构。

| 指标 | 含义 | 主要来源 | 证据类型 |
|---|---|---|---|
| `application_count` | 申报项目数 | `Sb_Jbxx` | 观测 |
| `requested_special_funding` | 申报专项经费 | `Sb_Jfgs` | 观测 |
| `funded_count` | 立项项目数 | `Ht_XMLXXX` 等立项事实 | 观测 |
| `contract_special_funding` | 合同专项经费 | `Ht_Jfgs` | 观测 |
| `avg_award_size` | 平均立项/合同规模 | 立项与合同经费聚合 | 观测聚合 |
| `active_institution_count` | 活跃机构数 | 项目-机构关系 | 观测聚合 |
| `active_person_count` | 活跃人员数 | 项目-人员关系、图谱 | 部分观测 |
| `review_score_proxy` | 评审强度代理 | `PS_XMPSXX` | 代理 |
| `topic_centrality` | 主题在结构中的中心性 | 图谱 | 代理 |
| `collaboration_density` | 协作密度 | 图谱 | 代理 |

### 2. 看变化：变化指标

用于识别哪些结构已经开始移动。

| 指标 | 含义 | 证据类型 |
|---|---|---|
| `application_yoy` | 申报量同比变化 | 观测聚合 |
| `funded_yoy` | 立项量同比变化 | 观测聚合 |
| `funding_yoy` | 经费同比变化 | 观测聚合 |
| `institution_entry_share` | 新进入机构占比 | 观测聚合 |
| `person_entry_share` | 新进入人员占比 | 部分观测 |
| `organization_concentration_delta` | 机构集中度变化 | 观测聚合 |
| `centrality_delta` | 主题中心性变化 | 代理 |
| `collaboration_delta` | 协作密度变化 | 代理 |
| `migration_inflow / outflow` | 热点迁入/迁出强度 | 代理 |

### 3. 看未来：基线预测指标

用于表达“如果什么都不改，接下来会怎样延续”。

| 指标 | 含义 | 形成方式 | 证据类型 |
|---|---|---|---|
| `baseline_heat_direction` | 升温/回落/持平 | 历史状态 + 变化信号 | 推演 |
| `baseline_funding_direction` | 经费扩张/收缩方向 | 历史结构延续 | 推演 |
| `baseline_structure_direction` | 集中/分散/迁移趋势 | 历史结构延续 + 图谱 | 推演 |
| `baseline_attention_level` | 下年度需关注程度 | 多指标综合排序 | 推演 |

注意：

- 这里的“未来”是条件性基线判断，不是确定性预言。
- 预测值必须附带证据窗口、样本量和不确定性声明。

### 4. 看风险：风险指标

风险指标不是单一指标，而是多类风险信号。

| 风险类型 | 典型信号 | 当前支撑强度 |
|---|---|---|
| 过热扩张风险 | 申报快速上升但立项/合同未同步 | 强 |
| 结构空心化风险 | 主题热度上升但活跃机构/人员收缩 | 中 |
| 集中度风险 | 少数机构长期占据过高份额 | 强 |
| 协作衰减风险 | 协作密度持续下降 | 中 |
| 迁移外流风险 | 相邻主题持续吸走项目和主体 | 中 |
| 人才接续风险 | 人员活跃度下降、新进入不足 | 弱到中 |

## 四、证据边界

每类指标都必须标注证据边界，至少分为四类：

| 标签 | 含义 |
|---|---|
| `observed` | 可由项目业务表直接观测 |
| `proxy` | 由评分、图谱或结构指标代理 |
| `derived` | 基于观测聚合形成 |
| `inferred` | 基于基线延续规则得到的条件判断 |

领导可见输出必须明确：

- 哪些结论来自真实业务事实
- 哪些是图谱代理
- 哪些是基线延续判断
- 哪些只是外部辅助信号，不应被当作硬事实

## 五、当前数据条件下的表达边界

基于当前“项目库 + 图谱”的真实底座，趋势模块可以比较扎实回答：

- 哪些主题在扩张、回落、迁移
- 哪些主题存在经费与热度的背离
- 哪些主题的机构格局和协作结构在变化
- 哪些主题在基线世界下更可能继续升温或累积风险

当前不能直接下硬结论的内容包括：

- 真实验收率、真实转化率
- 全省真实人才供给缺口
- 强因果结论
- 单靠外部信息就改写主结论

因此，趋势预判的本质不是“知道一切”，而是在真实数据边界内，把可观测变化、结构风险和基线未来说清楚。
