# 趋势预判分析链路

## 一、总链路

趋势预判建议采用如下链路：

```text
项目库 + 图谱
  -> topic 口径统一
  -> topic_time_panel 构建
  -> signal 检测
  -> migration 提取
  -> ranking 计算
  -> 标准化输出
```

## 二、阶段拆分

### 1. Panel Build

先统一主题口径和时间字段，再聚合形成 `topic_time_panel`。

这一层要解决两个问题：

- 各数据源中的 `topic` 如何归一
- 各年度上的状态指标如何稳定聚合

### 2. Signal Detect

这一层负责识别已经发生的变化。

输出的不是自然语言结论，而是结构化 signal，例如：

- 申报数显著上升
- 立项数连续下降
- 资金强度减弱
- 协作密度下降
- 中心性上升

### 3. Migration Extract

这一层负责从图谱中抽取主题迁移边和迁移强度。

重点不是构造复杂图算法名词，而是回答两个问题：

- 热点从哪里流向哪里
- 哪些主题处在迁移通道上

### 4. Trend Rank

这一层将状态和变化汇总成可消费的排序结果。

排序建议包括：

- 升温排序
- 降温排序
- 风险排序
- 迁移活跃排序

## 三、方法原则

### 1. 先状态，后变化

没有稳定状态面板，就不要先做预测叙事。

### 2. 先证据，后解释

每条 signal 和 ranking 都必须能回到原始指标。

### 3. 先排序，后预测

趋势预判先给出有证据支撑的趋势排序，再决定是否进一步做数值预测。

## 四、当前原型可复用部分

旧原型中可迁入趋势预判的部分包括：

- 热点迁移图算法
- 图谱结构信号提取
- 宏观风险规则中的结构化部分

但这些内容都应挂入 `signal_detect` 或 `migration_extract`，不再保留 step 命名。
