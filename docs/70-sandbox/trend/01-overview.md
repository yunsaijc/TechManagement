# 趋势预判总览

## 一、定位

趋势预判不是过去时统计，而是对 `Baseline World` 的未来延续判断。

它回答的问题是：

- 如果什么都不改，未来 1-2 个年度周期会怎么演化
- 哪些主题会继续升温、回落、迁移、空心化、风险累积

## 二、分析对象

趋势预判最终围绕四类对象展开：

- `Topic`
- `Institution`
- `Person`
- `Portfolio`

其中主分析层为：

`topic × year`

必要时下钻：

- `topic × institution × year`
- `topic × person × year`
- `project_cohort_outcome`

## 三、核心问题

趋势预判主要回答：

1. 哪些主题未来一年继续升温，哪些会回落
2. 热点正在从哪些主题迁移到哪些主题
3. 哪些主题出现“热度上升，但质量/兑现没有跟上”的背离
4. 哪些主题出现机构集中、主体更新不足、协作减弱等结构风险

## 四、输入

趋势预判输入主要包括：

- 历史窗口
- 主题口径
- 筛选范围
- 比较窗口
- 观测/代理边界

必要时可引入外部信息增强解释，但不改变当前以项目库和图谱为主底座的事实边界。

## 五、输出

趋势预判输出包括：

- `snapshot`
  当前结构快照
- `signals`
  升温、回落、背离、风险信号
- `migration`
  热点迁移与结构转移
- `ranking`
  机会与风险排序
- `baseline_forecast`
  不干预时的短期未来判断

## 六、能力边界

当前数据条件下，趋势预判可以比较扎实做：

- 主题热度变化
- 立项/合同经费结构变化
- 主题迁移
- 协作变化
- 机构集中和结构性风险信号

当前不能直接装作能做：

- 基于外部产业、论文、专利、新闻的完整前瞻预测
- 强因果结论
- 真实验收率与真实转化率判断
- 全省级真实人才供给判断

## 七、LLM 在 Trend 中的职责

LLM 用于：

- 领导问题解析
- 指标与信号归纳
- 风险解释与简报生成
- 追问式证据组织

LLM 不用于：

- 直接计算 trend 核心指标
- 编造趋势证据
- 把代理信号改写成确定性结论

## 八、与 Simulation 的关系

趋势预判先形成 `Baseline World`，再交给沙盘推演使用。

没有可信 baseline，就没有可信反事实。
