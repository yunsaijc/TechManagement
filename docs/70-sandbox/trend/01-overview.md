# 趋势预判总览

## 一、目标

趋势预判用于回答一个更具体的问题：

在不施加政策干预的前提下，主题状态正在如何变化，哪些主题正在升温、降温、迁移或累积风险。

它不是“大而全的预测平台”，而是一条围绕 `topic × time_window` 展开的分析主线。

## 二、核心思路

趋势预判只做四件事：

1. 建立主题状态面板
2. 检测状态变化信号
3. 提取主题迁移关系
4. 输出趋势与风险排序

对应的数据主线是：

```text
项目库 + 图谱
  -> topic_time_panel
  -> signals / migration / ranking
  -> baseline world
```

## 三、核心问题

1. 当前各主题的规模、强度和结构状态是什么
2. 哪些主题在升温，哪些主题在降温
3. 热点正在从哪些主题迁移到哪些主题
4. 哪些主题正在累积值得关注的风险

## 四、标准输出

- `snapshot`
- `signals`
- `migration`
- `ranking`

所有输出都必须能追溯到具体 `topic`、时间窗、指标和证据。

## 五、能力模块

趋势预判建议拆为以下能力模块：

1. `panel_build`
2. `signal_detect`
3. `migration_extract`
4. `trend_rank`

## 六、与其他模块的关系

- 趋势预判为沙盘推演提供 baseline world
- 趋势预判先产出结构化结果，再交给简报或问答层消费
- 趋势预判不是旧 step 流水线的重命名，而是新的分析底座
