# 70 输出与结论边界

## 一、目的

本文件定义系统最终可以输出什么，以及哪些话当前不能说。

## 二、输出分层

系统输出必须分三层：

### 1. 事实输出

基于真实观测：

- 申报、评审、立项、合同、经费结构变化
- 主题迁移与协作结构变化
- 连续弱势或持续过热主题

### 2. 解释输出

基于规则、统计与图结构的机制解释：

- 为什么某主题被判为高热低效
- 哪些机构或主题在驱动热点迁移
- 哪些风险来自集中度、协作衰减或评分-立项错配

### 3. 推演输出

在明确政策假设下的条件性结果：

- baseline vs scenario 对比
- 受益/受损主题
- crowding-out
- 风险迁移
- 方案建议

## 三、领导可见的标准输出

最终领导界面应围绕以下对象组织：

- 当前局势
- 基线未来判断
- 风险与机会清单
- 可选政策方案
- 推演对比
- 资源挪腾代价
- 证据链
- 局限与边界

## 四、当前数据下可以说的结论

### 1. 可以直接说

- 某主题申报量连续上升或下降
- 某主题立项和合同经费结构变化
- 某主题在图谱中的迁移和协作变化
- 某主题存在“高热低效”信号
- 某方案相对 baseline 更可能改变哪些结构

### 2. 只能弱说

- “存在人才接续风险信号”
- “存在协作能力减弱风险”
- “存在结构性空心化迹象”

这类表述必须同时给出：

- 数据覆盖
- 样本量
- 代理说明

## 五、当前数据下不能直接说的结论

除非后续补齐真实业务事实表，否则当前不应直接生成以下结论：

- “真实验收率低”
- “真实转化率低”
- “全省人才断层”
- “某政策已经证明导致某结果”
- “某主题 ROI 低”

## 六、结果页必须显示的披露信息

所有领导页结果都必须显示：

- 覆盖年份
- 覆盖对象数
- 主题口径
- 样本充分性
- 数据缺口
- 观测/代理标签
- baseline 与 scenario 的区别
- 关键假设

## 七、Trend 与 Simulation 的边界

- `Trend`
  只负责说“看到了什么变化、哪些变化值得警惕或跟进”
- `Simulation`
  只负责说“在明确政策假设下，变化可能朝什么方向演化”

两者都不能说：

- “未来一定如此”
- “模型已经替领导做出最终决定”
