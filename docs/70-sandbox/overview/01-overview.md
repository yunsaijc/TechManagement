# 70 领域总览

## 一、定位

`docs/70-sandbox/` 目录用于承载科技管理系统中与“研判、预测、推演”相关的中高层能力设计。

这一层不再以旧的 `sandbox step1-5` 原型流水线作为主组织方式，而是按能力域拆分为两条主线：

1. `trend/`：趋势预判
2. `simulation/`：沙盘推演

## 二、两条主线

### 1. 趋势预判

趋势预判回答的是：

- 当前结构正在发生什么变化
- 如果不做政策干预，未来一段时间会如何演化
- 哪些主题、人才结构、转化路径正在累积风险

### 2. 沙盘推演

沙盘推演回答的是：

- 如果改变某项政策动作，会发生什么
- 不同政策组合的结果差异是什么
- 影响通过什么路径传导到项目、人才、产出和转化

## 三、关系

趋势预判提供“基线世界”的结构和演化判断；
沙盘推演在基线之上施加政策干预，比较干预前后的差异。

```text
历史数据 / 图谱 / 指标
  -> 趋势预判（baseline）
  -> 沙盘推演（intervention / counterfactual）
  -> 简报 / 报告 / 问答
```

## 四、文档边界

- `overview/`：顶层边界、术语和重构范围
- `trend/`：趋势预判的对象、指标、分析链路和输出接口
- `simulation/`：沙盘推演的政策模型、场景机制和输出接口

旧的 step 原型文档保留在 `archive/`，作为历史探索材料，不再作为主设计入口。
