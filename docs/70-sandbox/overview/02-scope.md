# 70 范围与边界

## 一、为什么要拆分

当前原型把“热点迁移、规则告警、领导简报、GraphRAG 问答”放在一条流水线里，容易出现两个问题：

1. 将展示层误当成分析核心
2. 将趋势分析与政策干预混为一类问题

因此本阶段的重构目标不是继续堆步骤，而是重新定义能力边界。

## 二、纳入范围

### 1. 趋势预判

纳入以下问题：

- 热点迁移
- 主题兴衰
- 人才梯队变化
- 转化效率变化
- 前瞻风险信号
- 未来窗口预测

### 2. 沙盘推演

纳入以下问题：

- 资助方向调整
- 配额或准入约束调整
- 评审权重变化
- 人才补强或协作促进政策
- 多政策组合比较

## 三、不纳入范围

以下内容不作为 `docs/70-sandbox/` 的分析本体：

- 领导简报排版
- 前端卡片展示
- 通用问答界面
- 临时脚本入口和 demo 流水线命名

这些能力属于交付层，应建立在趋势和推演结果之上。

## 四、与旧原型的关系

旧原型文档中的组件可按以下方式重新定位：

- 热点迁移相关能力：并入趋势预判
- 宏观规则研判：并入趋势预判
- 简报编排：降级为交付层
- GraphRAG 问答：降级为交付层
- 总编排器：不再作为主架构，只保留为上层用例入口
