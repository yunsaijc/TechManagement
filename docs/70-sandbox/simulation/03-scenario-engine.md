# 沙盘推演场景引擎

## 一、场景的定义

`Scenario` 是一组政策动作、时间设置和约束条件的组合。

建议统一如下结构：

```json
{
  "scenario_id": "scenario_a",
  "baseline_ref": "trend_baseline_2026q1",
  "interventions": [],
  "assumptions": [],
  "constraints": [],
  "evaluation_targets": []
}
```

## 二、场景引擎职责

场景引擎至少负责以下工作：

1. 加载基线状态
2. 施加政策动作
3. 根据假设更新中间变量
4. 生成反事实结果
5. 与基线结果做差异比较

## 三、推演层次

建议将推演拆为三层：

### 1. mechanism layer

表达政策动作如何影响中间变量，例如：

- 准入收紧影响申报量
- 人才支持影响团队质量
- 协作激励影响跨主题合作密度

### 2. state transition layer

表达中间变量如何改变主题、项目和转化状态。

### 3. evaluation layer

表达最终需要比较的结果：

- 主题结构变化
- 人才结构变化
- 转化率变化
- 风险变化

## 四、最低可行版本

第一版不一定直接上完整结构因果模型，但至少需要：

1. 显式中间变量
2. 显式干预参数
3. 显式基线与反事实对比

否则“推演”仍然只是规则改写。
